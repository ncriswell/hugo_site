Introduction

Some might say that doing a data-driven retrospective on a previous season as the current season starts is a little useless. Those people are probably right, but they should have done it themselves if they are so clever. I think that Sleeper’s API has been improved over the off season and now I can (maybe) start doing weekly recaps that include graphs, tables, and other nerd stuff. (See the What the Hell is an API section of the appendix if you want to know more about what that means. Simply put, Sleeper’s API lets me pull all the league and player stat data I will need to report on whatever we want to report on.) I wanted to test this out and figured a recap of 2018 would be a good way to do that.

Disclaimers are good to get out of the way early, so let’s do that now.

  • I am using a process to create this document where the code to pull and analyze the data is written in the same file as the text you are reading now. Magic, right? However, the spelling and grammar check for my code editor of choice is…not great. Coupled with the fact that I’m already effectively functionally illiterate, the chance for typos is high.

  • I have spot checked all of the results that I pulled from the API against the actual values on the Sleeper site. They are usually spot on, but I have found about 0.2% of scores that are slightly off. I don’t know how this could happen. Maybe it is something that will become clear as we go through the 2019 season and can check things in real-time.

  • The analysis will all be scripted once we get into the season, so I don’t anticipate a lot of work to type these up each week. But, I haven’t really figured out a few things I’d like to eventually incorporate that will automate some of the explanatory text/commentary. Thus, some of the early reports will be heavy on tables and graphs and light on explanation. We’ll leave the commentary to the commish. And, the content for a weekly update will be much different than a several months overdue season recap. What I’m saying is that the first couple of weeks are going to be rough and/or non-existent.

  • I feel like this platform of analysis (HTML doc) is good but not great. I have some other things in mind, but this is definitely the easiest way to present the results as I can combine the data gathering, data analysis, and explanation into a quickly reproducible result. So there is a chance I’ll try something different, but I’ll be going with the HTML approach for now.

  • The look and feel of this report is pretty minimal for now. I am not really a web developer by trade, so I have a lot to learn about making stuff like this more customized and nice looking. I think this project will be a good way for me to learn that kind of thing. Perhaps I’ll even improve my actual fantasy performance through this process. Who knows.

  • The legibility of some of the graphs is going to be a problem due to the league size. Do you know how hard it is to make a color pallet that has 14 colors that are different enough to tell apart in a graph? Anyways, I am going to steal fivethirtyeight.com’s graph themes until I decide on my own.

I think that covers it. I am a big nerd so making this was pretty fun. Hopefully all the code I’ve written to pull together the 2018 data will be easily adaptable to 2019. As (if?) you go through this document, you’ll see that we are mostly looking at summary stats. I think it would be fun to tinker around a bit with some more predictive work. The API contains literally every stat that contributes to overall player score and also some crazy stuff like game time temperature and humidity. We could really go nuts with this. If there is some kind of analysis you would want to see, please send me an email or text, and I’ll see what I can do.

Appendix

What the Hell is an API?

Sleeper continues to be the best fantasy platform, and its not even close. One of their many, many updates (apparently) was the inclusion of weekly player stats to their API end points. As someone who took one programming class around 15 years ago, you should take this with a grain of salt, but from what I can tell an Application Programming Interface is sort of this catch-all term that refers to things that let computers talk to each other. So Sleeper has set up read-only end points as URLs that I (or anyone) can write code to fetch data from. For instance, go to this link. Just try it, I’m telling you, its great. Those are the week 1 stats for all players for the 2018 season. I can write a program that takes that wall of text and parses it for the useful stuff. No other platform even comes close to having a (free) usable API like this.
